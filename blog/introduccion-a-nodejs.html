¿Que es Node?

La definición más simple que les puedo dar es que Node es la vía para crear aplicaciones web con Javascript. Si eres un desarrollador que ya estas familiarizado con algunas librerías como jQuery o UnderscoreJS para enriquecer páginas o aplicaciones web en el navegador, con Node podrás utilizar todo ese conocimiento acumulado que tienes sobre Javascipt pero esta vez en el lado del servidor.

Como se describe su página oficial “Es una plataforma construida sobre el motor Javascript V8 de Chrome para crear aplicaciones de forma fácil, rápida y escalable”. Anteriormente mencioné que Node nos permite correr Javascript en el servidor utilizando el motor V8 que desarrolla Google para que el navegador Chrome muestre las páginas en tu PC. De esta forma podremos programar el servidor de una manera similar a lo que ya conoces de Javascript en el cliente.

Aunque seguimos utilizando el mismo lenguaje hay algunas diferencias entre programar del lado del cliente y del servidor. Existen algunas características únicas que tiene Node que nos proporcionan algunas ventajas sobre otros lenguajes. Como estamos acostumbrados, Javascript es un lenguaje orientado a eventos lo que nos permite tomar acciones en el momento específico en que el usuario interactúa con la página como dar un clic o enviar un formulario. En el lado del servidor Node también tiene la posibilidad de manejar eventos, pero de una manera diferente ya que no tenemos un DOM, pero de la misma forma que esperamos a que el usuario ejecute alguna acción con Node escuchara eventos como la carga de datos de la Base de Datos o transferencia de archivos.

Una de las características más potentes de Node sobre otros lenguajes de servidor es que está diseñado para evitar esperar que las cosas pasen para responder. Esto hace una real diferencia con respecto a los demás servidores tradicionales, por ejemplo en PHP cuando hace una petición a la base de datos se quedará esperando y no realizará ninguna otra acción hasta que la se devuelvan los resultados. Lo anterior mencionado no sucede con Node ya que es capaz de realizar varias operaciones de manera simultánea y tomar acciones cuando cada una de estas haya terminado de ejecutarse.

Otra de las características en las que Node es realmente bueno es creando aplicaciones en tiempo real, donde se mantiene una conexión entre el cliente y el servidor y una transferencia de datos en tiempo real. Esto permite que aplicaciones como los chats, servidores de juegos, de transmisión y acceso a datos.

Eso no es todo, Node es capaz realizar muchas cosas más y es lo que se tratará a lo largo de esta serie de publicaciones sobre NodeJS.

Como obtenerlo

Existen varias vías para obtener NodeJS y la principal es a través de su sitio web http://nodejs.org donde siempre encontrarás la última versión de su compilación para todas las plataformas disponibles. En este artículo abarcaré su uso sobre Windows x64. En la página oficial encontrarás las instalaciones pre compiladas para 32 y 64 bits de Windows listas para descargar e instalar.

Si no deseas utilizar una versión pre compilada y quieres compilar tu propio Node en la wiki del proyecto https://github.com/joyent/node/wiki/Installation puedes encontrar los pasos pare hacerlo.

La tercera vía por la que podemos obtenerlo es a través del gestor de paquetes de Windows llamado Chocolatey. De esta forma podremos instalar Node a través de la consola de Windows.

Instalación

Si optaste por descargar la versión compilada, una vez lo tengas ya en tu PC ejecutas el instalador y recorre los pasos hasta el final de la instalación.

En caso de que prefieras utilizar el gestor de paquetes de Windows Chocolatey teniéndolo previamente instalado en el sistema ejecutas el comando

choco install nodejs.install

Después de tenerlo instalado en el sistema podemos verificar si está listo para su uso. Para comprobarlo abrimos una consola de Windows y ejecutamos el comando.

node -v

El cual debe devolver un mensaje con la versión de Node que tenemos instalado como aparece en la siguiente imagen

En caso de que hayas instalado a través de Chocolatey también deberías poder ejecutar el comando anterior. Si al ejecutar el comando no obtenemos la versión de Node como respuesta quizás es que el comando node no está disponible. Esto se debe a que no se ha registrado en las variables de entorno del Sistema Operativo. Para resolver este problema te sugiero que leas el artículo Agregando Node en la consola de Windows.

Ahora ya tenemos NodeJS instalado en el sistema y listo para comenzar a usarlo.

Gestor de paquetes npm

NodeJs en si está pensado para realizar tareas de servidor, pero que pasa cuando queremos crear una aplicación web, en el momento nos damos cuenta que no está preparado para tales fines. Pero no es un problema, cuando instalamos Node en el sistema este vino acompañado por npm el gestor de paquetes de Node. Este gestor será el encargado de obtener todo tipo de paquetes que necesite nuestra aplicación para su funcionamiento.

A lo largo del uso de NodeJS para crear aplicaciones del lado del servidor haremos un uso extensivo de la herramienta npm para obtener paquetes como ExpressJs, Socket.io, mongoose y muchos más. Para comprobar que tenemos npm instalado podemos ir hasta la consola de Windows y ejecutar el siguiente comando.

npm -v

Si todo está bien obtendremos una respuesta con la versión de npm que tenemos instalado en el sistema como la imagen que se muestra a continuación.

Conclusiones

Si has realizado los pasos de manera correcta creo que no debes tener problemas para comenzar a utilizar NodeJS. A partir de este momento tienes una nueva vía para hacer uso de tus habilidades con Javascript y comenzar a construir tus servidores utilizando el mismo lenguaje que ya conoces. A medida que comiences a crear tus propios proyectos te darás cuenta porque Node es el lenguaje con más rápido crecimiento desde su creación y por qué la comunidad está tan activa con respecto a él.

Si tienes alguna duda con respecto a lo mencionado en este artículo no dudes en preguntar dejando un comentario, siempre estoy en la mejor disponibilidad de aclarar las dudas mientras tenga el conocimiento. Si esta publicación te ha parecido útil de alguna forma para tu aprendizaje, compártela con tus amigos y no olvides regresar a por más.
